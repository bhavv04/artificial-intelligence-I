%%%%% SECTION: summary
%%%%% Summarize your results in 5-10 sentences. Describe which states you tested on, 
%%%%% which ones ran in a reasonable amount of time (and how long they took), and 
%%%%% how much speedup you saw when using declarative heuristics. Report any other 
%%%%% interesting behaviour you saw.

We tested our planner with: 
    init state 1 -> goal state 11-15
    init state 2 -> goal state 21-22
    init state 3 -> goal state 31

Comparing regular vs. heuristic:
States 11-14 ran pretty much instantaneously, but looking at the times, we were still 
getting 2-10x faster times with heuristics enabled. State 15 took way too long on regular
mode but was done quite fast in heuristic mode (less than 20 seconds real time).

State 21 had the same results as states 11-14, running pretty quickly but still seeing a massive
improvement with heuristics. State 22 took way too long on regular mode so it was skipped. It still took
quite long with heuristics, almost 8 mins real time.

State 31 was only run with heuristics, but it ran quite a bit faster than state 22 with heuristics, despite
a similarly sized optimal sequence and a seemingly more complex goal state.

Our heuristics were able to cut down the solve time by such a great amount because we were able to
prune useless branch on our "state tree" so to speak.

For goal states 15, 22, and 31, we didnt put the logs because they were taking 
way too long to complete.
